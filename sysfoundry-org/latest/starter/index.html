Getting Started

You can skip steps 1 & 2 if you already have "Docker Desktop" & "Git" in your environment.

Before we delve into details, lets look at Sysfoundry in action.

1. Setup Docker Desktop for your OS

The Sysfoundry Dev environment relies on the Docker for Desktop tool. You can access Docker Desktop for your OS here

Once you have installed the Docker Desktop tool on your machine, configure the Docker Desktop tool to change the resources to use atlease 4 core & 4 GB RAM. You can find the relevant documentation from Docker. For example the one for Mac OS is here

2. Setup Git for your OS

Setup Git from here

3. Configure your hosts file

The sysfoundry base dev environment requires your hosts configuration to point "keycloak" to your local host ip "127.0.0.1"

You can find instructions on how to do this for your relevent os here.

4. Checkout Sysfoundry Base Dev Deployment scripts

Once above mentioned steps are done, run the following command from a folder on your machine. Assume for the purposes of this document this folder is being referred to as SYSFOUNDRY_ORG_HOME.

git clone https://github.com/sysfoundry-org/deployment.git

5. Startup Sysfoundry Base

cd $SYSFOUNDRY_ORG_HOME/deployment/dev/solutions/sysfoundry-dev-base/
docker-compose up -d

If everything goes well, you should be able to access the Sysfoundry Base environment with the following URL

http://localhost:9090/wicket/

You should be redirected to your IAM Login as below for a fictional "XYZ-SOLUTION".

Enter the below credentials

Username : user1

Password : user1

Once you login, you will be landing on the App Dashboard as shown below

6. Shutdown Sysfoundry Base

Once you playaround / explore the Sysfoundry Base environment , you can bring down the whole environment with the below simple command.

docker-compose down
